Build a self-hosted hardware fleet inventory dashboard for a broadcast company's edit systems. Windows workstations push their hardware specs to this app via a REST API, and IT staff view/filter/flag them in a web dashboard for upgrade budget planning. This is an internal NOC-style tool — data density and clarity matter more than marketing polish.
Stack
Backend: Python FastAPI + SQLAlchemy 2.x, PostgreSQL (use Replit's built-in Postgres).
Frontend: React + Vite + TypeScript, served by the backend or proxied. Tailwind for styling.
Auth: username/password login with sessions prep for authentic sso.

Data model
One row per machine — current state only, upserted on each report (no history table).
`machines` table:
`machine_id` (string, PK) — a stable GUID generated by the collector client, NOT the serial (serials are often "Default string" on whitebox boards).
`hostname` (string, indexed)
`site` (string, nullable, indexed) — e.g. Dallas / Tustin / Nashville
`last_seen` (timestamptz)
Promoted/filterable columns: `manufacturer`, `model`, `cpu`, `total_ram_gb` (int), `ram_type` (string e.g. DDR4/DDR5), `gpu1_model`, `os`, `primary_ip` (string — the machine's main IPv4)
`data` (JSONB) — the full raw spec payload
API endpoints
`POST /api/report` — ingest endpoint. Protected by a bearer token (a shared secret read from an env var `INGEST_TOKEN`; reject with 401 if `Authorization` header != `Bearer <token>`). Upserts by `machine_id`, sets `last_seen` to now. Accepts this JSON body:
```json
  {
    "machine_id": "guid", "hostname": "EDIT-01", "site": "Dallas",
    "manufacturer": "MSI", "model": "MS-7D86",
    "cpu": "Intel(R) Core(TM) i9-14900K | 24C/32T @ 3.2GHz",
    "total_ram_gb": 128, "ram_type": "DDR5",
    "gpu1_model": "NVIDIA GeForce RTX 4090",
    "os": "Microsoft Windows 11 Pro 10.0.26100", "primary_ip": "10.20.5.42",
    "data": {
      "Manufacturer": "...", "Model": "...", "Serial": "...",
      "Motherboard": "...", "BIOS": "...", "OS": "...", "CPU": "...",
      "TotalRAM_GB": 128, "RAM_Type": "DDR5", "RAM_Slots": "4 used / 4 total",
      "RAM_Modules": "32GB DDR5 DIMM Corsair ... ; 32GB DDR5 DIMM Micron ...",
      "GPU1_Model": "...", "GPU1_Driver": "...", "GPU2_Model": "...", "GPU2_Driver": "...",
      "Disks": "Samsung 980 PRO [SSD] 1863GB SCSI ; ...",
      "Volumes": "C: 1862GB (1074GB free) ; ...",
      "NICs": "Intel I226-V 954Mbps [10.20.5.42] ; Intel Wi-Fi 6E AX211 160MHz ...",
      "PrimaryIP": "10.20.5.42"
    }
  }
  ```
Note: multi-value fields inside `data` (RAM_Modules, Disks, Volumes, NICs) are single strings with items joined by `" ; "` — split on that in the UI to render as lists.
`GET /api/machines` — list all, each row including a computed `flags` array (see below). Login required.
`GET /api/machines/{machine_id}` — full detail incl. raw `data`. Login required.
`DELETE /api/machines/{machine_id}` — delete a machine. Login required + bearer token OR admin session.
`GET /api/health` — DB connectivity check, no auth.
`GET /api/export.csv` — download all machines as a flat CSV (for handing specs to finance). Login required.
Upgrade flags (computed server-side per machine)
Return a `flags` string array. Tune thresholds in one clearly-marked function:
`ram_type` starts with "DDR3" → "DDR3 (old platform)" (danger)
`ram_type` == "DDR4" → "DDR4" (warn)
`total_ram_gb` < 64 → "Low RAM (NGB)" (warn)
`gpu1_model` does NOT contain any of RTX 40 / RTX 50 / RTX 60 / RTX PRO / RTX A → "GPU upgrade candidate" (warn)
`os` contains "Windows 10" → "Windows 10 (EOL Oct 2025)" (danger)
`last_seen` older than 14 days → "Stale (Nd)" (danger). Handle timezone-naive vs aware datetimes defensively — coerce naive to UTC before subtracting, or it throws.
Authentication
Local accounts in a `users` table: `username`, `password_hash` (bcrypt/argon2), `role` ("admin" | "viewer").
On first boot, if no users exist, seed an admin from env vars `ADMIN_USER` / `ADMIN_PASSWORD` and print a notice.
Session-cookie login (`POST /api/login`, `POST /api/logout`, `GET /api/me`). HTTP-only, secure cookies.
All `/api/machines*` and `/api/export.csv` routes require a valid session. `/api/report` uses the bearer token instead (machines don't log in). `DELETE` requires admin role.
Login page: clean, centered card matching the dark theme below.
Admin-only "Users" screen to add/remove accounts and reset passwords.
Visual design
Dark operations-console aesthetic. The data is the hero — no hero banners, no marketing copy.
Palette: background `#0b0e14`, panels `#121722` / `#161c2a`, borders `#232c3d`, text `#d6deec`, muted `#7d8aa3`. Accent teal `#36d0c4`. Status colors: ok green `#4ad07a`, warn amber `#e0b341`, danger red `#e0556b`.
Type: monospace UI throughout (ui-monospace / JetBrains Mono / Menlo). Uppercase letter-spaced section labels.
Top bar: "Fleet Inventory" wordmark with a teal period accent, subtitle "OBTV Edit Systems", current logged-in user + logout on the right.
Stat strip: small cards — total machines, flagged count (amber), site count.
Main table: sortable columns (click header toggles asc/desc with a teal ▲/▼) for Host, IP, Site, CPU, RAM, Type, GPU, OS, Seen, Flags. Host in teal bold. Flags render as small rounded pills colored by severity; a clean machine shows a green "clean". Row hover highlight; click a row to open a detail drawer.
Toolbar above table: free-text search across host/IP/CPU/GPU/model/OS; a site dropdown filter; an "All / Flagged only" dropdown; an "Export CSV" button.
Detail drawer (slides in from the right, closes on Esc or backdrop click): machine hostname + model header, flag pills, a key/value block (manufacturer, model, serial, motherboard, BIOS, OS, primary IP, CPU, RAM summary, GPU1 + driver, GPU2 + driver), then titled sections that split the `" ; "`-joined strings into line lists: Memory Modules, Disks, Volumes, Network Adapters (each NIC line includes its IPv4 in brackets).
Dashboard auto-refreshes the machine list every 30s. Responsive down to tablet width; visible keyboard focus; respect reduced-motion.
